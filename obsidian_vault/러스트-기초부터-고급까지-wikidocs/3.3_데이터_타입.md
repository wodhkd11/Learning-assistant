# 3.3 데이터 타입

데이터 타입은 변수 혹은 상수에 어떤 성격의 데이터가 할당되는 지를 결정하는 속성이다. Rust는 정적 타입(Statically Typed) 언어이다. 모든 변수/상수의 타입이 컴파일 시점에 결정이 되어야한다는 의미이다. Rust는 정적 타입 언어이기에 선언 시점에 변수의 타입이 결정될 수 있어야 한다. 이 말이 선언 시점에 타입을 꼭 명시해야한다는 말은 아니다. let b=1; 처럼, 변수의 타입을 컴파일 타임에 결정할 수 있기만하면 된다. 이 경우 변수 b의 타입은 i32가 된다. 컴파일러의 타입 추론에 의해서 i32 타입으로 자동 지정된다. 1이라는 것이 정수이고 이 경우 i32로 처리하도록 컴파일러의 로직에 되어 있어서, 컴파일 타임에 b의 타입을 i32로 설정할 수 있는 것이다.
let a:u32 = 1;  //타입을 u32로 명시했다.
let b = 1;        //타입을 명시하진 않았지만, 정수형인것을 알 수 있고, 이 경우 Rust는 i32타입으로 지정
let c = "abc"; // &str 타입이 된다.
let d = "2".parse().expect("not a number");  //에러 발생한다.
위 코드에서 마지막 d를 선언할 때 에러가 발생한다. "2".parse().expect()에 의해 d의 타입을 결정할 수 없기 때문이다. 즉, i32인지 u32인지 등을 결정할 수 없다는 얘기다. 프로그래머가 아래와 같이 타입을 결정해줘야 한다.
let d:u32 = "2".parse().expect("not a number");   // let d:u32 와 같이 변수 타입 결정해줌
let d = "2".parse::<u32>().expect("not a number");  // parse::<u32>() 와 같이 parse 메서드의 리턴타입을 결정해줌
"2".parse()는 문자열 "2"를 숫자로 바꾸는 메서드이다. 리턴 타입은 Result이다. Rust에서 사용되는 기본 데이터 타입(Primitive type)을 정리해 보자.
스칼라 타입: 정수, 부동 소숫점, 문자, 부울
복합 타입: 튜플, 배열
